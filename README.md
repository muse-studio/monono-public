# Monono

## 目的

福知山公立大学情報学部橋田研究室の備品の貸出や管理等を行うためのウェブアプリ。

### 備考

ゼミ生は最大でもせいぜい20人程度であり、管理する備品の数も100は超えないと思われる。  
そのため、スプレッドシートなどのデータ量が膨大になることはまずない。  
大企業のような規模の管理システムと同じように考える必要はない。

## 使い方

### 一般ユーザ

できること：

- 一覧：ゼミ内の備品の在庫一覧を確認できる。
- 貸出：必要事項を記入し、備品を借りられる。
- 利用中：自身が現在借りている備品一覧を確認できる。
- 履歴：過去の貸出履歴を閲覧できる。

### 管理者

- 在庫：ゼミ内の備品の在庫一覧の閲覧・備品情報の編集・在庫の新規追加ができる。
- 貸出：貸し出された記録一覧を閲覧できるほか、特定のユーザ（主に学生）に貸出の登録ができる。
- ユーザ：Mononoに登録されているユーザ（学生や教員等）の一覧を閲覧できる。
- 新規登録：必要事項を記入し、Mononoにユーザを新規登録できる。
- 設定：各種設定（GASを用いるにあたり、データのバックアップや一括インポート等の操作は接続元のGoogle Spreadsheetから直接行うことになった）。
- 監査：監査ログを確認できる。

## 構造

詳しくはコードを読んで欲しい。

### フロントエンド

一般的なhtmlで記述。Github Pagesを用いて公開。

### バックエンド

Google Apps Script（GAS）を使用。リポジトリにある`code.gs`はそのコピー。  
ここのコードを変更した際はブラウザのGAS上のコードも上書きすべし。

## データ管理方式

PagesとGASを組み合わせて実装したところ、毎回データを取得してきてページに反映する方式だとデータのロード等に一々5～10秒かかって非常に煩わしかった。  
そのため、データは3段階に分け、それぞれのフェーズで同期させていくことで、擬似的に高速なページのロードを行えるようにしている。

### スプレッドシートのデータ

クラウド上に存在し、最も正確。ここにあるデータを元にしてキャッシュは構成される。  
ただし、データを取得するのに毎回リダイレクト等が発生し、ただのテキストデータを取ってくるのにも10秒くらいかかる。遅い。

### GASのキャッシュ

クラウド上に存在する、スプレッドシートからGASが取得してきたデータのコピー。  
スプレッドシートが更新されるとここのキャッシュも更新される。  
ネットワークを介すので読み込みに1～2秒かかる。やや遅い。

### デバイス内のローカルストレージ

ローカルデバイスに保存されている、GASのキャッシュのコピー。  
GASのキャッシュに変更があれば上書きされる。
これをデバイスが読み取ってページに反映するのは爆速。ラグはほとんど考慮する必要はない。

## 安全性

スプレッドシートにはパスワードはハッシュとして暗号化されて記録されるので、セキュリティ的には問題はない...はず。  
また、パスワード以外の情報については、ゼミ生であればこのアプリを介さずとも容易に入手が可能なものが多いと考えられるため、逐次削除したり厳密に管理するよりかは、利便性のために保存されていて構わないと思っている。
